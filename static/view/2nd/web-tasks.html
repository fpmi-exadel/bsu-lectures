Общее Задание. "Реализовать приложение для управления списком пользователей и их авторизацией"

Требуется создать web приложение(Serlvet/JSP) для управления списком пользователей с поддержкой авторизации пользователя средствами Basic Authentication | 1 | 2 |

Требования

Картинка описывает общие требования к пользовательскому интерфейсу

ПО должно поддерживать два типа пользователей с соответствующими правами:

  • администратор — может создавать, редактировать информацию и удалять пользователей
  • пользователь — может просматривать таблицу пользователей. При попытке обращения к запрещенному функционалу пользователь должен попадать на страницу "Access Denied"
Упрощенный вариант задания - Задача может быть реализована без модуля авторизации, т.е. необходимо реализовать только редактирование списка пользователей, без разделения прав доступа и формы авторизации.

Указания и рекомендации

На следующей диаграмме вы можете увидеть примерную структуру приложения. Это не требование, это рекомендация

Рекомендуемая последовательность выполнения задачи:

  1. Сверстать необходимые страницы и реализовать проверку полей ввода данных. Инструменты для разработки – HTML, CSS, Javascript
  2. Создать структуру каталогов для веб приложения:
  3. Реализовать необходимые классы сервлетов и обеспечить переходы между страницами
  4. Реализовать на страницах отображение динамических данных из Request Scope – список пользователей, единичный пользователь. Набор данных на данном этапе можно жестко предопределить в сервлетах.
  5. Реализовать сервисный класс UserService и класс доступа к данным UserDAO. Предопределенные данные перенести в DAO.
  6. Создать хранилище данных в памяти используя HashMap<ID пользователя, ссылка на объект User>. Предопределенные данные убрать.
  7. Создать хранилище данных в СУБД (MySQL, Postgres или любая другая) – таблица USER c колонками id (автоинкрементное поле), email, password, first_name, last_name, role
  8. В каталог Tomcat/lib подложить JDBC драйвера БД
  9. Переписать DAO с использованием JDBC для доступа к БД.
  10. Реализовать механизм авторизации пользователей определяемых в таблице БД средствами Tomcat. Реализовать разграничение прав доступа пользователей.

Задание 2. "Контактная информация"

Добавить к Заданию 1 возможность вводить контактную информацию пользователя.

Администратор может редактировать всех пользователей

Обычный пользователь может просматривать информацию других пользователей и редактировать только свою контактную информацию.

Оценка

  1. ПО работает как указано в задании 1..3 балла
  2. Использование СУБД (Postgres, MySQL, MS SQL и т.д.) данных 3 балла
  3. Ответы на вопросы по Tomcat 1..5 баллов
  4. Ответы на вопросы по СУБД 1..5 баллов
