Общие рекомендации/требования к заданиям

  1. В качестве сервера рекомендуется использовать Tomcat 7
  2. Все сообщения пользователю, надписи и др. должны быть локализуемыми, например используя properties файлы.
  3. Для хранения данных можно использовать xml. Для работы с xml можно использовать jaxb
    • http://www.mkyong.com/java/jaxb-hello-world-example/ - простые и понятные примеры
    • https://jaxb.java.net/tutorial - исчерпывающий tutorial
  4. Чистота вашего кода и хорошее проектирование будут значительно влиять на процесс оценки приложения

Задание 1. "Web Файлообменник"

Требуется создать web приложение(Serlvet/JSP) для обмена файлами (простая версия https://rapidshare.com/ ). Приложение должно поддерживать следующие возможности:
  • Обычные пользователи
    1. Загружать файл(до 5 Mb) на сервер – сделать отдельную страничку, на которой имеются поля ввода информации о файле(название, описание, автор, примечания). Предусмотреть проверку корректности вводимых данных, т.е. все необходимые поля заполнены и т. д. Если поля ввода содержат некорректные данные, то подсветить их красным цветом. После успешной загрузки файла показать пользователю сообщение и ссылку на скачивание/удаление файла. Например:
      • http://localhost:8080/myapp/files/id12345
      • http://localhost:8080/myapp/files/id12345?op=remove&code=Sj34Z
      Параметр code является аналогом пароля подтверждающего правомерность удаления файла.
    2. Скачивать файл по заданной ссылке в которой содержится идентификатор файла, например http://localhost:8080/myapp/files/id12345. Если идентификатор ошибочный - показать соответствующую страницу.
    3. Удалять свои файлы используя ссылку, полученную при загрузке файла на сервер.
    4. Просматривать список файлов, включая информацию о файле и его размер(предусмотреть отдельную странчку). Список файлов и их описания вывести в виде таблицы
  • Модераторы(требуется ввод логина-пароля для входа в систему)
    1. Просматривать список файлов на сервере
    2. Скачивать и загружать файлы как и другие пользователи
    3. Удалять любой файл

Задание 2. "Телефонный справочник Web"

Требуется создать web приложение(Serlvet/JSP) реализующее функцию хранения и поиска в телефонном справочнике.
Приложение должно поддерживать следующие возможности:
  1. Обычные пользователи
    • Просматривать список телефонов, включая ФИО и адрес(город, улица, дом, корпус, квартира). Данные вывести в виде таблицы.
    • Поиск в телефонной книге по номеру,части ФИО или адреса.
  2. Модераторы(требуется ввод логина-пароля для входа в систему)
    • Добавлять новые и редактировать существующие данные.
    • Удалять любой файл

Задание 3. "Галерея картинок"

Требуется создать web приложение(Serlvet/JSP) реализующее функцию хранения картинок(JPG,PNG).
Приложение должно поддерживать следующие возможности:
  • Обычные пользователи
    1. Загружать файл(до 5 Mb) на сервер – сделать отдельную страничку, на которой имеются поля ввода информации о файле(название, описание, автор, теги примечания). Предусмотреть проверку корректности вводимых данных, т.е. все необходимые поля заполнены и т. д. Если поля ввода содержат некорректные данные, то подсветить их красным цветом. После успешной загрузки файла показать пользователю сообщение и ссылку на скачивание/удаление файла. Например:
      • http://localhost:8080/myapp/shared/img123
      • http://localhost:8080/myapp/shared/img123?op=remove&code=Sj34Z
      Параметр code является аналогом пароля подтверждающего правомерность удаления картинки.
    2. Просматривать изображение по заданной ссылке, в которой содержится идентификатор файла, например http://localhost:8080/myapp/shared/img123. Если идентификатор ошибочный - показать соответствующую страницу.
    3. Просматривать детальную информацию об изображение по заданной ссылке, в которой содержится идентификатор файла, например http://localhost:8080/myapp/shared/img123/details. Если идентификатор ошибочный - показать соответствующую страницу. На данной странице показать уменьшенное изображение(например, 100 x 100) и информацию о картинке.
    4. Удалять свои файлы используя ссылку, полученную при загрузке файла на сервер.
    5. Просматривать список файлов, включая информацию о картинках и его размер(предусмотреть отдельную странчку). Список картинок вывести в виде таблицы состоящей из уменьшенных изображений(например, 100 x 100)
  • Модераторы (требуется ввод логина-пароля для входа в систему)
    1. Добавлять новые и редактировать существующие данные.
    2. Удалять любой файл

Уменьшенные картинки должны создаваться приложением без участия пользователя. Для работы с изображениями можно использовать классы

  • java.awt.image.BufferedImage
  • javax.imageio.ImageIO

Задание 4. "График функции (Web)"

Требуется создать web приложение(Serlvet/JSP) реализующее функцию построения графика функции y=f(x).
Пример http://www.wolframalpha.com/input/?i=plot+sin%281%2Bx%5E2%29
Приложение должно поддерживать следующие возможности:
  1. Ввод функции. Функция представляет собой аналитическое выражение состоящее из чисел, аргумента x, арифметических операций (+,-,*,/), возведения в степень, скобок, и не менее 5 простейших “встроенных” функций(например, sin, cos, exp, log, abs). При вводе некоректного значения приложение должно вывести сообщение пользователю.
  2. Построение графика функции в виде картинки. Отобразить координатные оси и график функции разными цветами.
  3. Выбор интервала для построения графика функции, при этом приложение должно корректно работать даже если в выбранном интервале есть точки, где функция не определена.
  4. Предусмотреть историю вводимых функций(например список на странице).
Note:
  • Для вычисления значения функции можно использовать обратную польскую запись
  • Для построениея графика функции как картинки на стороне сервера можно использовать следующие классы
    • java.awt.image.BufferedImage
    • javax.imageio.ImageIO

Задание 5. "Каталог Библиотеки(Web)"

Требуется создать web приложение(Serlvet/JSP) реализующее функцию каталога библиотеки.
Приложение должно содержать следующие страницы:
  1. Список книг – таблица содержащая информацию о книге(название, автор, дата издания, издательство, isbn, описание, ключевые слова/теги). Каждая строка должна содержать ссылку на страницу редактирования и добавления новой книги.
  2. Поиск – поиск карточек по всем атрибутам(см. п. 1.)
  3. Редактирование – строница содержащая поля ввода инофрмации о книге(см. п. 1.), кнопки:
    • Сохранить(обновить данные)
    • Отмена (отмена изменений и возвта)

Предусмотреть проверку корректности вводимых данных:

      Название – обязательное поле, произвольный текст длиной до 300 символов
      Автор – обязательное поле, произвольный текст длиной до 100 символов
      Дата издания – обязательное поле, 4 цифры
      Издательство – обязательное поле, произвольный текст, длиной до 100 символов
      Isbn – обязательное поле, текст формата “ISBN xxxxxxxxxxxxx“ (13 цифр, которые могут быть разделены дефисом)
      Описание – опциональное поле, текст длиной до 500 символов
      Ключевые слова/теги - опциональное поле, набор слов состоящих из русских и(или) английских слов, цифр, символа подчёркивания. Слова разделены пробелом, общая длина поля не должна превышать 150 символов
Замечание: данная страница должна использоваться и как страница создания новой карточки о книге
